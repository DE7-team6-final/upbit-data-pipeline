# Commit Convention

Conventional Commits 기반의 최소 규칙입니다.

## Format
<type>: <message> (#issue-number)

## Types
- `feat`: 새로운 기능 추가
- `fix`: 버그 수정
- `docs`: 문서 변경
- `refactor`: 기능 변화 없는 코드 개선
- `config`: 환경/설정/CI 변경

## Examples
feat: add websocket reconnect logic (#12)
fix: consumer timestamp bug (#33)
docs: update architecture diagram (#5)
